### 1. Цель тестирования
```
  - Проверить корректность работы классов и методов Restaurant:
     
  - Добавление и удаление еды в чеке;
     
  - Проверка суммы заказа;
     
  - Корректный запуск приложения.
```

### 2. Объекты тестирования
```
  | Компонент | Результат |
  |:------:|:------:|
  | Main | Главный класс системы заказов ресторана |
  | Menu | Класс с меню еды и напитков |
  | MenuItem | Класс с хранением информации о позиций в меню |
  | CafeOrder | Класс с реализацией системы заказов|
```

### 3. Типы тестов
```
  | Тип теста | Цель |
  |:------:|:------:|
  | Модульное тестирование | Проверка отдельных методов классов |
  | Интеграционное тестирование | Проверка взаимодействия между классами |
  | Функциональное тестирование | Проверка соответствия программе требованиям |
  | Нагрузочное тестирование | Проверка устойчивости при большом количестве еды в заказе |
```
